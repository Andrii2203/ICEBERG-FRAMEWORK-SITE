The Only Valid Structure for the 6 Audit Files Generated by Claude
Цей стандарт визначає єдину допустиму структуру, яку Claude повинен використовувати при генерації 6 файлів UI‑аудиту.
Claude не має права змінювати, додавати, видаляти або переставляти секції.

Це структурний закон, а не рекомендація.

1. UI_UX_AUDIT.md
Purpose
Глибокий аналіз UI‑екрана: логіка, структура, компоненти, поведінка, проблеми.

Structure
# UI/UX Audit

## 1. Screen Overview
- Короткий опис того, що видно на екрані
All findings must be based strictly on visible UI elements.
No assumptions about hidden states or backend behavior.
- Основні блоки
- Основні функції

## 2. Layout Structure
- Опис макета (header, sidebar, main, footer, модалки)
- Просторова організація
- Вирівнювання

## 3. Component Analysis
- Опис кожного компонента
- Його роль
- Його стан
- Його поведінка
If an element is unclear or unreadable, Claude must explicitly mark it as “uncertain”.
## 4. Interaction Model
- Які дії може виконати користувач
- Які елементи інтерактивні
- Які стани можливі

## 5. UX Issues
- Проблеми
- Причини
- Наслідки

## 6. Recommendations
- Конкретні покращення
- Пріоритети

2. COMPONENT_MAP.md
Purpose
Повний список компонентів, які видно на UI‑зображенні.

Structure
# Component Map

## 1. Component List
- Назва компонента
- Тип (button, input, card, table, nav, modal, toast, icon, badge)
- Роль
- Стан (default, hover, active, disabled, error, loading)

## 2. Component Groups
- Primary components
- Secondary components
- Layout components
- Interactive components
- Informational components

## 3. Notes
- Виявлені патерни
- Повторювані елементи
- Неконсистентності

6. audit.json
Purpose
Структурований JSON, який бекенд може використовувати для автоматизації.

Structure

{
  "screen_overview": "...",
  "components": [
    {
      "name": "",
      "type": "",
      "role": "",
      "states": []
    }
  ],
  "issues": [
    {
      "id": "",
      "severity": "low | medium | high | critical",
      "description": "",
      "recommendation": ""
    }
  ],
  "hierarchy": {
    "primary": [],
    "secondary": []
  },
  "score": {
    "ui_quality": 0,
    "ux_quality": 0,
    "accessibility": 0,
    "overall": 0
  }
}

Data in audit.json must directly reflect findings in markdown files with no contradictions.
