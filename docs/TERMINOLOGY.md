# TERMINOLOGY / GLOSSARY
Unified glossary of terms for the Iceberg Framework Landing Website.

## 1. CORE CONCEPTS
- **Iceberg Framework:** A deterministic system of standards, protocols, and execution maps designed to control AI‑assisted development.
- **Determinism:** Every action, rule, and process produces predictable, reproducible results.
- **STOP‑CHECK:** A mandatory validation point where progress must pause until conditions are met.

## 2. PROJECT SPECIFIC TERMS
- **Parallax Iceberg:** A visual representation of the framework using layered SVG or CSS elements that move at different speeds on scroll.
- **Visible Tip (UI/Docs):** The part of the website explaining the user-facing side of Iceberg.
- **Submerged Mass (Logic/Protocols):** The part of the website explaining the internal core of Iceberg.

## 3. AUDIT SPECIFIC TERMS
- **UI Detection:** The process of classifying an uploaded image as `ui`, `non-ui`, or `chaos`.
- **Chaos Mode:** A state where the system detects a messy or unstructured UI, leading to a "Free Audit" using Groq.
- **Full Audit:** A premium analysis of a valid UI interface using Claude 3.5 Sonnet, resulting in a 6-file professional package.

## 4. COMMERCIAL TERMS
- **Solo Pack (Starter Pack):** The base commercial offering containing `rules_normalized.yaml` and the initial Context Matrix for individual use.
- **Agency License:** A team-tier offering allowing up to 50 developers to use Iceberg assets across multiple client projects.
- **Deterministic Matrix:** The core logical engine (binary or YAML) that provides the "Submerged Mass" logic to an AI model.
- **Signed URL (TTL):** A temporary, secure link for downloading assets from R2, valid for exactly 10 minutes.
